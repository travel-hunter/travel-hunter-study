# 파이썬 가상환경 학습 정리

## 1. 가상환경의 의미

파이썬에서 말하는 **가상환경**은 실체가 없는 가짜 환경이 아니다.

가상환경은 실제로 디렉터리로 생성되는 **격리된 Python 실행/패키지 환경**이다.

```text
가상환경 = 프로젝트별로 분리된 Python 작업 공간
```

즉, “가상”이라는 표현은 “존재하지 않는다”는 뜻이 아니라, **시스템 전체 환경과 분리되어 있다**는 의미에 가깝다.

---

## 2. 가상환경을 사용하는 이유

가상환경을 사용하는 가장 큰 이유는 **프로젝트별 패키지 충돌을 막기 위해서**이다.

예를 들어 프로젝트마다 필요한 패키지 버전이 다를 수 있다.

| 프로젝트   | 필요한 패키지       |
| ------ | ------------- |
| A 프로젝트 | FastAPI 0.100 |
| B 프로젝트 | FastAPI 0.110 |

이 패키지들을 시스템 전체에 설치하면 버전 충돌이 날 수 있다.

그래서 프로젝트마다 독립된 가상환경을 만들어 패키지를 따로 관리한다.

```text
가상환경을 쓰는 이유 = 프로젝트별 패키지 버전 충돌 방지
```

---

## 3. 가상환경은 실제 디렉터리로 만들어진다

가상환경을 생성하면 실제 디렉터리가 생긴다.

```bash
python3 -m venv venv
```

예시 구조:

```text
project/
├── venv/
└── app.py
```

여기서 `venv/`가 **가상환경 디렉터리**이다.

---

## 4. 가상환경 디렉터리 안에 들어가는 것

가상환경 디렉터리에는 프로젝트에서 사용할 Python 실행 파일, pip, 설치된 패키지 등이 들어간다.

예시:

```text
venv/
├── bin/
├── lib/
├── include/
└── pyvenv.cfg
```

패키지를 설치하면 보통 다음 위치에 저장된다.

```text
venv/lib/python3.x/site-packages/
```

---

## 5. 가상환경은 디스크 용량을 차지한다

가상환경은 실제 디렉터리이므로 디스크 저장공간을 차지한다.

특히 패키지를 많이 설치할수록 가상환경 디렉터리의 크기가 커진다.

| 패키지 예시   | 용량 영향  |
| -------- | ------ |
| requests | 작음     |
| fastapi  | 비교적 작음 |
| numpy    | 큼      |
| pandas   | 큼      |
| torch    | 매우 큼   |

중요한 점은 가상환경이 주로 차지하는 것은 **RAM 메모리**가 아니라 **디스크 저장공간**이라는 것이다.

```text
가상환경 = 디스크에 생성되는 실제 디렉터리
```

---

## 6. 가상환경은 삭제 후 다시 만들 수 있다

가상환경 디렉터리는 보통 소스코드처럼 반드시 보관해야 하는 대상이 아니다.

삭제해도 다시 만들 수 있다.

```bash
rm -rf venv
```

다시 생성:

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

따라서 보통 `venv/` 자체는 Git에 올리지 않는다.

대신 필요한 패키지 목록은 `requirements.txt`에 저장한다.

```bash
pip freeze > requirements.txt
```

---

## 7. 가상환경 디렉터리는 기본적으로 숨겨지지 않는다

다음처럼 만들면 `venv/`는 일반 디렉터리이다.

```bash
python3 -m venv venv
```

`ls` 명령어로 확인할 수 있다.

```bash
ls
```

출력 예시:

```text
app.py  venv
```

---

## 8. 숨김 디렉터리로 만들 수도 있다

Linux/macOS에서는 이름 앞에 `.`을 붙이면 숨김 디렉터리처럼 동작한다.

```bash
python3 -m venv .venv
```

일반 `ls`에는 보이지 않는다.

```bash
ls
```

하지만 `ls -a`로 보면 확인할 수 있다.

```bash
ls -a
```

출력 예시:

```text
.  ..  .venv  app.py
```

---

## 9. 일반 디렉터리와 숨김 디렉터리의 차이

일반 디렉터리와 숨김 디렉터리의 본질적인 차이는 **이름**이다.

| 구분      | 예시       | 특징             |
| ------- | -------- | -------------- |
| 일반 디렉터리 | `venv/`  | 기본 `ls`에서 보임   |
| 숨김 디렉터리 | `.venv/` | 기본 `ls`에서 안 보임 |

숨김 디렉터리라고 해서 더 안전하거나, 권한이 다르거나, 암호화되는 것은 아니다.

```text
숨김 디렉터리 = 이름이 .으로 시작해서 기본 목록에서 숨겨지는 일반 디렉터리
```

---

## 10. Docker 안에서도 가상환경을 만들 수 있다

Docker 컨테이너 안에서도 Python 가상환경을 여러 개 만들 수 있다.

```bash
python3 -m venv venv1
python3 -m venv venv2
```

하지만 보통 실무에서는 Docker 컨테이너 자체가 이미 격리 환경이기 때문에, 컨테이너 안에서 여러 개의 가상환경을 만드는 경우는 많지 않다.

```text
Docker = 애플리케이션 실행 환경 격리
venv = Python 패키지 환경 격리
```

비유하면 다음과 같다.

```text
Docker는 집을 따로 만드는 것
venv는 그 집 안에 방을 따로 만드는 것
```

---

## 최종 핵심 요약

```text
파이썬 가상환경은 프로젝트별로 Python 패키지를 격리하기 위한 실제 디렉터리 기반 환경이다.
```

더 짧게 정리하면 다음과 같다.

```text
가상환경 = 프로젝트 전용 Python 패키지 공간
```

반드시 기억할 핵심은 세 가지이다.

| 핵심     | 설명                                 |
| ------ | ---------------------------------- |
| 실체가 있음 | `venv/` 또는 `.venv/` 디렉터리로 실제 생성됨   |
| 격리 목적  | 프로젝트별 패키지 충돌을 막음                   |
| 재생성 가능 | `requirements.txt`를 이용해 다시 만들 수 있음 |
